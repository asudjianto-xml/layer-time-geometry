\subsection*{Who this book is for}

This book is for data scientists, ML engineers, and anyone who wants to
understand \emph{what happens inside a transformer} when it processes text.
You know what a neural network is. You have used PyTorch or TensorFlow. You
may have fine-tuned a language model. But you have probably never asked: what
does the internal state of the model \emph{look like}, mathematically, as it
moves from layer to layer?

That is the question this book answers. It is a practical, hands-on companion
to the research monograph, replacing formal proofs with intuition, code, and
visualizations.

\subsection*{What you need}

\begin{itemize}[nosep]
\item \textbf{Linear algebra}: matrix multiplication, eigenvalues, SVD.
  If you can multiply two matrices and know what an eigenvector is,
  you are ready.
\item \textbf{Basic statistics}: mean, variance, PCA, ANOVA
  (at the ``I've seen it in a stats class'' level).
\item \textbf{Python/PyTorch}: enough to load a model and run a forward pass.
\item \textbf{Calculus}: partial derivatives and gradients.
  Nothing beyond what you see in machine learning courses.
\end{itemize}

You do \emph{not} need differential geometry, measure theory, geometric
algebra, or operator theory. The research monograph that this book is based on
uses all of those. This book translates the key ideas into language you already
speak.

\chapter{Opening the Black Box}
\label{chap:intro}
% ══════════════════════════════════════════════════════════════════════════════

\begin{whycare}
You use language models every day---for code completion, summarisation,
question answering. But when a model hallucinates, how do you diagnose the
failure? When a model ignores your instructions, where in the computation did
things go wrong? To answer these questions, you need to see inside.
\end{whycare}

\section{The question}

A transformer language model takes a sequence of tokens and processes them
through a stack of layers. At each layer, every token has a
\textbf{hidden state}---a high-dimensional vector that encodes everything the
model ``knows'' about that token at that point in the computation.

The central question of this book is:

\begin{center}
\large\itshape\color{Navy}
What is the geometric structure of these hidden states,\\
and what does it tell us about the model's computation?
\end{center}

\section{Hidden states: the model's internal workspace}

\begin{analogy}
Think of a transformer like a factory assembly line. Raw materials (tokens)
enter at one end. At each station (layer), workers (attention heads and
feed-forward networks) modify the product. The hidden state at each
station is like a detailed spec sheet describing the product's current
state---its shape, color, material, intended use. By the time the product
reaches the end of the line, the spec sheet contains everything needed to
produce the final output.

This book is about reading those spec sheets and understanding what the
workers are doing at each station.
\end{analogy}

Concretely, if a model has $L$ layers and processes $T$ tokens, then at
layer~$l$ and token position~$t$, the model holds a vector:
\[
  H(l, t) \in \mathbb{R}^p
\]
where $p$ is the model's hidden dimension (e.g., $p = 4096$ for a 7B
parameter model). The collection of all these vectors forms a
two-dimensional grid:

We call this the \textbf{layer--time grid}. ``Layer'' is the vertical axis
(depth in the network). ``Time'' is the horizontal axis (position in the
sequence). Every cell holds a $p$-dimensional vector.

\begin{keyidea}
The hidden states of a transformer form a \textbf{field} on a
two-dimensional grid. The geometry of this field---how vectors change across
layers and across tokens---reveals the model's internal computation.
\end{keyidea}

\section{The four big ideas}

This book develops four main ideas, each building on the last:

\begin{enumerate}
\item \textbf{Standardize the vectors} (Chapter~\ref{chap:whitening}).
  Raw hidden states have correlated, differently-scaled dimensions.
  We fix this with PCA-based whitening, like standardizing variables
  before running a regression.

\item \textbf{Measure how vectors change} (Chapters~\ref{chap:kernels}--\ref{chap:curvature}).
  We decompose the layer-to-layer transformation into a rotation
  (direction change) and a stretch (magnitude change), then measure
  curvature---how much the grid's geometry twists.

\item \textbf{Run controlled experiments} (Chapter~\ref{chap:experiments}).
  We use factorial experimental design (DOE) to test which input
  properties actually affect the geometry: task type? reasoning
  difficulty? prompt length?

\item \textbf{Connect geometry and dependency to reasoning-like computation}
  (Chapter~\ref{chap:reasoning}).
  We add a new observable---\emph{dependency}---that measures how much
  each layer's hidden state matters for the model's final output.
  Combining geometry with dependency lets us give practical operational
  definitions of reasoning, memory, and control.
\end{enumerate}

\chapter{Standardizing the Internal Representation}
\label{chap:whitening}
% ══════════════════════════════════════════════════════════════════════════════

\notebook{ch2\_whitening.ipynb}

\begin{whycare}
If you compare hidden states at different layers without standardizing them
first, you are comparing apples and oranges. Whitening is to hidden-state
analysis what z-scoring is to statistical analysis: it removes scale
differences and correlations so that the geometry you measure is real, not an
artefact of the coordinate system.
\end{whycare}

\section{The problem with raw hidden states}

A transformer's hidden state vector has $p$ dimensions (typically 2048--8192).
These dimensions are \emph{not} independent:
\begin{itemize}[nosep]
\item Some dimensions carry much more variance than others.
\item Many dimensions are correlated.
\item The scale changes from layer to layer.
\end{itemize}

If you directly compute distances between hidden states at different layers,
the distance is dominated by whichever dimensions happen to have the largest
variance---not necessarily the most informative ones.

\begin{analogy}
Imagine measuring the similarity between cities using two features: population
(in millions) and average temperature (in degrees). Population ranges from
0.01 to 30; temperature ranges from $-20$ to $45$. Without standardisation,
Euclidean distance is almost entirely determined by population, and temperature
barely matters. The same problem occurs in hidden states, but with thousands
of dimensions.
\end{analogy}

\section{The solution: PCA-based whitening}

The fix is a standard statistical tool: \textbf{whitening via PCA}.

\textbf{Step 1: Center.} Subtract the mean hidden state $\bar{H}$ computed
across all layers and tokens for a given input.

\textbf{Step 2: Compute the covariance.} The $p \times p$ covariance matrix
$\Sigma$ captures how dimensions co-vary.

\textbf{Step 3: PCA projection.} Take the top $k$ eigenvectors
$V_k = [v_1, \ldots, v_k]$ of $\Sigma$, with eigenvalues
$\lambda_1 \geq \cdots \geq \lambda_k > 0$.

\textbf{Step 4: Whiten.} Define the whitening matrix
$W = V_k \,\text{diag}(\lambda_1^{-1/2}, \ldots, \lambda_k^{-1/2})$
and compute:
\[
  \widetilde{H}(l,t) = \bigl(H(l,t) - \bar{H}\bigr) \, W
  \quad \in \mathbb{R}^k
\]

The result: $\widetilde{H}$ has uncorrelated dimensions with unit variance.
We use $k = 256$ throughout---this retains the meaningful variation while
making computation tractable.

\begin{keyidea}
After whitening, Euclidean distance in the $k$-dimensional space equals
Mahalanobis distance in the original $p$-dimensional space. This means the
geometry we measure respects the model's internal metric, not an arbitrary
coordinate choice.
\end{keyidea}

\section{Why this matters: metric consistency}

The core theoretical result (proven formally in the research monograph) is:

\begin{tcolorbox}[base, colback=LGreen, colframe=DGreen]
\textbf{Metric Consistency Theorem (informal).}
After whitening, the covariance of the hidden states is the identity matrix.
All subsequent geometric operations---decompositions, distances,
curvatures---are invariant to the original coordinate system of the model.
\end{tcolorbox}

In plain language: whitening lets us compare geometry \emph{across layers,
across prompts, and across models} in a principled way. Without it, we would
be measuring artefacts.

\section{Python implementation}

\begin{lstlisting}[caption={Whitening hidden states from a Hugging Face model}]
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

def extract_and_whiten(model, tokenizer, text, k=256):
    """Extract whitened hidden states from a transformer."""
    inputs = tokenizer(text, return_tensors="pt").to(model.device)

    with torch.no_grad():
        out = model(**inputs, output_hidden_states=True)

    # Stack all layers: shape (L+1, T, p)
    H = torch.stack(out.hidden_states)     # includes layer 0
    H = H[1:]                               # skip embedding layer
    L, T, p = H.shape

    # Step 1: Centre
    H_flat = H.reshape(-1, p)               # (L*T, p)
    H_mean = H_flat.mean(dim=0)
    H_centred = H_flat - H_mean

    # Step 2-3: PCA via SVD
    U, S, Vt = torch.linalg.svd(H_centred, full_matrices=False)
    V_k = Vt[:k].T                          # (p, k)
    eigenvalues = (S[:k] ** 2) / (L * T - 1)

    # Step 4: Whiten
    W = V_k * (1.0 / eigenvalues.sqrt())    # (p, k)
    H_whitened = H_centred @ W              # (L*T, k)
    H_whitened = H_whitened.reshape(L, T, k)

    return H_whitened  # shape: (L, T, k)
\end{lstlisting}

\section{What we skip on first reading}

The research monograph proves several additional properties: that whitening
is the unique linear transformation achieving metric consistency, that the
effective rank of the covariance determines the intrinsic dimensionality,
and that the choice of $k$ is stable for $k \geq 128$.

\begin{pitfall}
\textbf{Do not skip layer 0.} Layer~0 is the raw token embedding before any
transformer processing. Including it in the whitening distorts the geometry
because it has a fundamentally different distribution. Always start from
layer~1. The \texttt{ltg} library handles this automatically.
\end{pitfall}

\chapter{Measuring Similarity: Kernels on the Grid}
\label{chap:kernels}
% ══════════════════════════════════════════════════════════════════════════════

\notebook{ch3\_kernels.ipynb}

\begin{whycare}
Kernels answer the most basic question about the hidden-state grid: which
layers are similar to each other? Which token positions are similar? The
answers reveal the model's computational phases---setup, processing,
and decision-making.
\end{whycare}

\section{What is a kernel?}

A kernel is just a similarity function. Given two objects $a$ and $b$, a
kernel $K(a, b)$ returns a number indicating how similar they are. You already
know several kernels:

\begin{itemize}[nosep]
\item \textbf{Dot product}: $K(a, b) = a^\top b$ --- the simplest kernel.
\item \textbf{Cosine similarity}: $K(a, b) = \frac{a^\top b}{\|a\|\|b\|}$
  --- normalized dot product.
\item \textbf{RBF kernel}: $K(a, b) = \exp(-\|a - b\|^2 / 2\sigma^2)$
  --- similarity decays with distance.
\end{itemize}

We define two kernels on the layer--time grid.

\section{The layer kernel: which layers are similar?}

For each layer $l$, collect all $T$ whitened token vectors into a matrix
$\widetilde{H}^{(l)} \in \mathbb{R}^{T \times k}$. The \textbf{layer kernel}
measures how similar two layers are:
\[
  K_{\text{layer}}(l, l') = \frac{1}{T}
  \operatorname{tr}\!\left(
    \widetilde{H}^{(l)\top} \widetilde{H}^{(l')}
  \right)
\]
This is just the average dot product between corresponding token vectors at
two layers.

\begin{analogy}
Think of each layer as a photograph of the assembly line taken from above.
The layer kernel measures how much two photographs look alike. Early layers
look similar to each other (setup phase), late layers look similar to each
other (decision phase), and there is often a visible transition in the
middle (processing phase).
\end{analogy}

\section{The temporal kernel: which tokens are similar?}

Similarly, for each token $t$, collect its representation across all layers:
$\widetilde{H}_t \in \mathbb{R}^{L \times k}$. The \textbf{temporal kernel}
measures:
\[
  K_{\text{time}}(t, t') = \frac{1}{L}
  \operatorname{tr}\!\left(
    \widetilde{H}_t^\top \widetilde{H}_{t'}
  \right)
\]

\section{What the kernels reveal: three computational phases}

\begin{finding}
When we compute the layer kernel for typical prompts across multiple models
(Qwen2.5-7B, DeepSeek-R1-7B, Qwen2.5-32B), we consistently find
\textbf{three phases}:

\begin{enumerate}[nosep]
\item \textbf{Early layers} ($l \lesssim L/4$): High mutual similarity.
  The model is building up basic representations.
\item \textbf{Middle layers} ($L/4 \lesssim l \lesssim 3L/4$): Rapid change.
  This is where the ``work'' happens---attention heads route
  information, representations diverge.
\item \textbf{Late layers} ($l \gtrsim 3L/4$): Convergence toward a
  prediction-ready state. Curvature peaks here.
\end{enumerate}
\end{finding}

\begin{lstlisting}[caption={Computing the layer kernel matrix}]
def layer_kernel(H_whitened):
    """Compute the L x L layer kernel matrix.

    Args:
        H_whitened: tensor of shape (L, T, k)
    Returns:
        K: tensor of shape (L, L)
    """
    L, T, k = H_whitened.shape
    # Reshape to (L, T*k) and compute normalised Gram matrix
    H_flat = H_whitened.reshape(L, T * k)
    K = H_flat @ H_flat.T / T
    return K
\end{lstlisting}

\chapter{How Information Flows: Operators and Polar Decomposition}
\label{chap:operators}
% ══════════════════════════════════════════════════════════════════════════════

\notebook{ch4\_polar\_decomposition.ipynb}

\begin{whycare}
This is the chapter where the framework becomes powerful. We decompose
\emph{how} hidden states change from one layer to the next into two
fundamental operations: rotation and stretching. This decomposition is
the key to understanding what each layer actually does.
\end{whycare}

\section{The layer transition operator}

Between layer $l$ and layer $l+1$, the hidden states change. We can
approximate this change with a linear operator $T^{(l)}$ that best maps
$\widetilde{H}^{(l)}$ to $\widetilde{H}^{(l+1)}$:
\[
  \widetilde{H}^{(l+1)} \approx \widetilde{H}^{(l)} \, T^{(l)}
\]

We find $T^{(l)}$ by ordinary least squares---the same linear regression
you learned in your first statistics course, just applied to matrices:
\[
  T^{(l)} = \left(\widetilde{H}^{(l)\top}\widetilde{H}^{(l)}\right)^{-1}
  \widetilde{H}^{(l)\top}\widetilde{H}^{(l+1)}
\]

\begin{analogy}
If the hidden states at layer $l$ are the ``input data'' and the hidden
states at layer $l+1$ are the ``output data'', then $T^{(l)}$ is the
best linear predictor. It tells us: what linear transformation best
explains how the representation changed?
\end{analogy}

\section{Polar decomposition: rotation + stretch}

Every square matrix can be decomposed as:
\[
  T^{(l)} = U^{(l)} \, P^{(l)}
\]
where:
\begin{itemize}[nosep]
\item $U^{(l)}$ is an \textbf{orthogonal matrix} (rotation/reflection):
  it changes directions without changing magnitudes.
\item $P^{(l)}$ is a \textbf{symmetric positive-definite matrix}
  (stretching): it scales different directions by different amounts.
\end{itemize}

This is called the \textbf{polar decomposition}, and it is analogous to
writing a complex number as $z = e^{i\theta} \cdot r$ (angle + magnitude).

\begin{keyidea}
Every layer transition in a transformer does two things:
\begin{enumerate}[nosep]
\item \textbf{Rotate} the representation (change which directions are
  active), captured by $U^{(l)}$.
\item \textbf{Stretch} the representation (amplify some directions,
  compress others), captured by $P^{(l)}$.
\end{enumerate}
These are the two fundamental primitives of transformer computation.
\end{keyidea}

\section{What rotation and stretching mean}

\begin{center}
\renewcommand{\arraystretch}{1.5}
\begin{tabular}{@{}L{2cm} L{4cm} L{4.5cm}@{}}
\toprule
\textbf{Component} & \textbf{What it does} & \textbf{Computational meaning} \\
\midrule
$U^{(l)}$ (rotation) &
  Rotates the coordinate axes; preserves all distances &
  Attention mixing: redistributes information across directions without
  losing or amplifying anything \\
$P^{(l)}$ (stretch) &
  Stretches some directions, compresses others &
  Feature selection: amplifies important directions, suppresses
  irrelevant ones \\
\bottomrule
\end{tabular}
\end{center}

\section{Measuring the stretch: eigenvalues of P}

The eigenvalues $\sigma_1 \geq \sigma_2 \geq \cdots \geq \sigma_k$ of
$P^{(l)}$ tell us how much each principal direction is stretched:

\begin{itemize}[nosep]
\item $\sigma_i > 1$: direction $i$ is amplified (the model is paying
  more attention to this direction).
\item $\sigma_i < 1$: direction $i$ is compressed (the model is
  discarding information along this direction).
\item $\sigma_i = 1$: direction $i$ is unchanged.
\end{itemize}

The \textbf{condition number} $\kappa = \sigma_1 / \sigma_k$ measures how
selective the layer is. A high condition number means the layer strongly
amplifies some directions while crushing others---it is making a sharp
``decision''.

The \textbf{effective rank} $\text{erank} = \exp(H(\hat\sigma))$, where
$H$ is the entropy of the normalized eigenvalue distribution, measures how
many directions are actively used. Low effective rank means the layer
concentrates its computation in a few directions.

\begin{lstlisting}[caption={Polar decomposition of layer transitions}]
def polar_decompose_layers(H_whitened):
    """Compute polar decomposition T = U @ P for each layer.

    Args:
        H_whitened: (L, T, k) whitened hidden states
    Returns:
        Us: list of L-1 orthogonal matrices (k, k)
        Ps: list of L-1 symmetric positive-definite matrices (k, k)
    """
    L, T, k = H_whitened.shape
    Us, Ps = [], []

    for l in range(L - 1):
        H_l = H_whitened[l]       # (T, k)
        H_next = H_whitened[l+1]  # (T, k)

        # Least-squares transition operator
        T_l = torch.linalg.lstsq(H_l, H_next).solution  # (k, k)

        # Polar decomposition
        U, S, Vt = torch.linalg.svd(T_l)
        P = Vt.T @ torch.diag(S) @ Vt   # symmetric positive-definite
        U_orth = U @ Vt                   # orthogonal
        Us.append(U_orth)
        Ps.append(P)

    return Us, Ps
\end{lstlisting}

\chapter{Where the Action Is: Curvature}
\label{chap:curvature}
% ══════════════════════════════════════════════════════════════════════════════

\notebook{ch5\_curvature.ipynb}

\begin{whycare}
Curvature tells you \emph{where} in the layer--time grid the interesting
computation happens. Zero curvature means the model could be replaced by
a simple linear transformation at that point. High curvature means something
genuinely non-trivial is occurring. This is the single most useful
diagnostic in the framework.
\end{whycare}

\section{The idea of curvature on a grid}

In everyday life, curvature measures how much a surface bends. A flat table
has zero curvature. A basketball has constant positive curvature. A saddle
has negative curvature.

On the layer--time grid, curvature measures something analogous: how much
the geometry ``twists'' when you go around a small loop.

\begin{analogy}
Imagine walking around a square block in a city. You make four right turns
and end up facing the same direction---zero curvature. Now imagine walking
the same square on the surface of a sphere (like walking a small square near
the North Pole). After four right turns, you \emph{do not} face the same
direction---the surface's curvature rotated you. The same idea applies to
hidden states: go right one token, go up one layer, go left one token,
go down one layer. If you end up with a different transformation than the
identity, there is curvature.
\end{analogy}

\section{Computing curvature on the layer--time grid}

Consider a small square (``plaquette'') at position $(l, t)$ on the grid:

\begin{center}
\begin{tikzpicture}[scale=1.3]
  \draw[thick, Navy] (0,0) rectangle (2,2);
  \node[below left, font=\small] at (0,0) {$(l,t)$};
  \node[below right, font=\small] at (2,0) {$(l,t\!+\!1)$};
  \node[above right, font=\small] at (2,2) {$(l\!+\!1,t\!+\!1)$};
  \node[above left, font=\small] at (0,2) {$(l\!+\!1,t)$};
  \draw[-{Stealth}, thick, Blue] (0.1, 0.15) -- (1.9, 0.15)
    node[midway, below, font=\scriptsize] {$T^{(l)}_{\text{time}}$};
  \draw[-{Stealth}, thick, DGreen] (1.85, 0.2) -- (1.85, 1.8)
    node[midway, right, font=\scriptsize] {$T^{(l)}_{\text{layer}}$};
  \draw[-{Stealth}, thick, DRed] (1.9, 1.85) -- (0.1, 1.85)
    node[midway, above, font=\scriptsize] {$(T^{(l+1)}_{\text{time}})^{-1}$};
  \draw[-{Stealth}, thick, Amber] (0.15, 1.8) -- (0.15, 0.2)
    node[midway, left, font=\scriptsize] {$(T^{(l)}_{\text{layer}})^{-1}$};
\end{tikzpicture}
\end{center}

Going around this loop gives a combined transformation:
\[
  \Omega(l, t) = T^{(l)}_{\text{time}} \cdot T^{(l)}_{\text{layer}}
  \cdot \left(T^{(l+1)}_{\text{time}}\right)^{-1}
  \cdot \left(T^{(l)}_{\text{layer}}\right)^{-1}
\]

If the grid were ``flat'' (i.e., moving right then up gives the same result
as moving up then right), then $\Omega(l,t) = I$. The deviation from
identity \emph{is} the curvature:
\[
  \text{curvature at } (l,t) = \|\Omega(l,t) - I\|_F
\]
where $\|\cdot\|_F$ is the Frobenius norm (think: sum of squared entries,
then square root).

\begin{keyidea}
Curvature measures \textbf{non-commutativity}: does it matter whether you
go ``right then up'' or ``up then right'' on the grid? If yes, the
computation at that point is genuinely nonlinear and non-trivial.
\end{keyidea}

\section{What curvature tells us}

\begin{finding}
Across all models tested (three architectures, two scales), curvature
consistently shows the same pattern:

\begin{itemize}[nosep]
\item \textbf{Early layers}: Low curvature. The model is doing simple,
  nearly-linear processing.
\item \textbf{Middle layers}: Moderate curvature. Information routing
  and mixing.
\item \textbf{Final 3--5 layers}: Curvature spikes sharply. This is
  where the model commits to its prediction.
\end{itemize}

This is the most robust finding in the entire framework---it replicates
across models, prompts, and tasks.
\end{finding}

\begin{pitfall}
\textbf{Curvature does not equal reasoning complexity.} Early in this
research, we conjectured that higher curvature meant harder reasoning.
Controlled experiments across three models showed this is \textbf{false}:
reasoning depth (number of logical steps) has no significant effect on
curvature. Curvature measures \emph{non-separable interaction} between
layers and tokens---not how ``hard'' the model is thinking.
\end{pitfall}

\chapter{Testing What Matters: Experimental Design}
\label{chap:experiments}
% ══════════════════════════════════════════════════════════════════════════════

\notebook{ch6\_experiments.ipynb}

\begin{whycare}
Anyone can compute a metric. The hard part is knowing whether the metric
actually tells you something about the input. Experimental design (DOE) is
the tool that separates real signal from noise. This chapter uses the same
factorial designs you learn in statistics courses---applied to transformer
geometry.
\end{whycare}

\section{The experimental setup}

We designed 320 prompts varying five factors:

\begin{center}
\renewcommand{\arraystretch}{1.4}\small
\begin{tabular}{@{}L{3cm} L{6cm}@{}}
\toprule
\textbf{Factor} & \textbf{Levels} \\
\midrule
Cognitive task & Arithmetic, Deductive Reasoning, Causal Reasoning,
  Linguistic Analysis, Factual Retrieval, Definition \\
Reasoning depth & Low (1-step), Medium (2-3 steps), High (4+ steps) \\
Prompt length & Short ($<$50 tokens), Medium, Long ($>$150 tokens) \\
Knowledge domain & Science, History, Everyday, Abstract \\
Robustness & Original, Paraphrased \\
\bottomrule
\end{tabular}
\end{center}

For each prompt, we extracted whitened hidden states and computed $\sim$20
geometric metrics (curvature, condition number, effective rank, dependency
measures, etc.). Then we ran ANOVA: which factors significantly affect which
metrics?

\section{Three models}

We ran the experiment on three models to check replicability:

\begin{center}
\renewcommand{\arraystretch}{1.3}
\begin{tabular}{@{}lll@{}}
\toprule
\textbf{Model} & \textbf{Parameters} & \textbf{Layers} \\
\midrule
Qwen2.5-7B & 7 billion & 28 \\
DeepSeek-R1-Distill-Qwen-7B & 7 billion & 28 \\
Qwen2.5-32B & 32 billion & 64 \\
\bottomrule
\end{tabular}
\end{center}

\section{The results: five key findings}

\begin{finding}
\textbf{R1: Task type matters, but modestly.}

Cognitive task type (arithmetic vs.\ reasoning vs.\ retrieval, etc.) is
\emph{detectable} in the geometry---certain dependency metrics differ
significantly across task types. But the effect sizes are small
($\eta^2 \approx 0.02$--$0.05$). Task type is a real signal, not the
dominant driver.
\end{finding}

\begin{finding}
\textbf{R2: Surface features are not negligible.}

We expected that ``shallow'' features like prompt length and formatting would
have no effect on geometry. \textbf{Wrong.} Prompt length affects dependency
horizons significantly, especially in DeepSeek-R1. Robustness condition
drives the largest single effect in Qwen-7B ($\eta^2 = 0.098$ for the
global curvature ratio). The geometry responds to both computational content
\emph{and} input form.
\end{finding}

\begin{finding}
\textbf{R3: Reasoning depth has no effect on geometry.}

This is the cleanest and most important negative result. Across all three
models, the factor ``reasoning depth'' (1-step vs.\ multi-step problems)
\emph{fails to reach significance} on any geometric metric. The one minor
exception: condition number in Qwen-7B ($\eta^2 = 0.032$, $p = 0.010$).

\textbf{What this means:} you cannot tell how many reasoning steps a problem
requires by looking at the geometry of the hidden states. The number of
logical steps does not map to any measurable internal geometric quantity.
\end{finding}

\begin{finding}
\textbf{R4: Curvature concentrates in the final layers.}

This finding replicates unanimously across all three models. In Qwen2.5-32B
(64 layers), peak curvature is entirely in the final 3 layers. This is the
most robust qualitative finding in the study.
\end{finding}

\begin{finding}
\textbf{R5: Dependency tracks task type more closely than curvature.}

Dependency-based metrics (how much each layer's state matters for the output)
are more sensitive to task type than curvature-based metrics. They are also

\chapter{What Each Layer Contributes: Dependency}
\label{chap:dependency}
% ══════════════════════════════════════════════════════════════════════════════

\notebook{ch7\_dependency.ipynb}

\begin{whycare}
Geometry tells you \emph{what shape} the hidden states have. Dependency tells
you \emph{which parts matter}. A layer might have complex geometry but
contribute nothing to the final output. Dependency distinguishes
``busy work'' from ``real work.''
\end{whycare}

\section{The idea}

After a model processes a prompt, it produces a score (log-probability) for
the next token. Some layers contributed heavily to this score; others barely
mattered. \textbf{Dependency} measures this contribution using gradients.

\begin{analogy}
In a relay race, some runners gain time and others lose time. The final
clock time depends on all runners, but not equally. Dependency is like
looking at the split times to figure out who actually made the difference.
\end{analogy}

\section{Computing dependency}

The dependency of layer $l$ is:
\[
  D_l = \left\|\frac{\partial s}{\partial \widetilde{H}^{(l)}}\right\|_F
\]
where $s$ is the model's score for the next token. This is just the
gradient of the output with respect to the hidden state at layer~$l$.
Layers where the gradient is large ``mattered more'' for the output.

\begin{lstlisting}[caption={Computing dependency profiles}]
def dependency_profile(model, tokenizer, text, k=256):
    """Compute dependency D_l for each layer."""
    inputs = tokenizer(text, return_tensors="pt").to(model.device)

    # Forward pass with gradient tracking
    out = model(**inputs, output_hidden_states=True)
    hidden_states = out.hidden_states[1:]  # skip layer 0

    # Score: log-prob of the predicted next token
    logits = out.logits[0, -1, :]          # last position
    score = logits.max()                    # score of top token

    # Backward pass
    score.backward()

    # Collect gradient norms per layer
    D = []
    for hs in hidden_states:
        if hs.grad is not None:
            D.append(hs.grad.norm().item())
        else:
            D.append(0.0)

    return D  # list of length L
\end{lstlisting}

\textbf{Note}: the code above is simplified. In practice, you need to enable
gradient computation for intermediate hidden states, which requires hooks.
See the research monograph's implementation appendix for the full version.

\section{Dependency profiles}

The dependency profile $D_0, D_1, \ldots, D_{L-1}$ reveals which layers
drive the output.

Useful summary statistics:
\begin{itemize}[nosep]
\item $D_{\text{total}} = \sum_l D_l$ --- total sensitivity to all layers.
\item \textbf{Dependency horizon} $H_\alpha$ --- the layer at which
  $\alpha\%$ of the total dependency is reached. A low $H_{90}$ means
  the model makes up its mind early.
\item \textbf{Dependency entropy} $S = -\sum_l \hat{D}_l \log \hat{D}_l$
  (where $\hat{D}_l = D_l / D_{\text{total}}$) --- how spread out the
  dependency is. High entropy = many layers contribute; low entropy = a
  few layers dominate.
\end{itemize}

\begin{pitfall}
\textbf{Length confound.} Total dependency $D_{\text{total}}$ is strongly
correlated with sequence length ($r \approx 0.96$). Longer sequences have
more hidden states for the gradient to flow through, mechanically increasing
$D_{\text{total}}$. Always use \textbf{normalized} metrics (entropy,
horizons, concentration ratios) when comparing prompts of different lengths.
\end{pitfall}

\chapter{Reasoning, Memory, and Control}
\label{chap:reasoning}
% ══════════════════════════════════════════════════════════════════════════════

\notebook{ch8\_reasoning\_memory\_control.ipynb}

\begin{whycare}
This chapter connects everything. The geometry and dependency tools from
earlier chapters combine to give \emph{practical operational definitions}
of what it means for a transformer to ``reason,'' ``remember,'' and be
``controlled.'' These are not claims that geometry alone explains
reasoning---they are measurable conditions that, when satisfied together,
indicate reasoning-like computation that you can detect and test.
\end{whycare}

\section{Defining reasoning geometrically}

We all have intuitions about what ``reasoning'' means, but we need a
precise definition to measure it. Here is ours:

\begin{keyidea}
A transformer is \textbf{reasoning} at layer $l$ and token $t$ when three
conditions hold simultaneously:
\begin{enumerate}[nosep]
\item \textbf{Interaction}: The curvature $\|\Omega(l,t)\|$ is
  significantly above baseline---the model is doing non-trivial,
  nonlinear processing.
\item \textbf{Propagation}: The metric factor $P^{(l)}$ has high condition
  number---the model is selectively amplifying some directions.
\item \textbf{Dependency}: The dependency $D_{l,t}$ is high---the
  processing at this point actually matters for the output.
\end{enumerate}
All three must be present. Interaction without dependency is ``busy work.''
Dependency without interaction is just passing information through.
\end{keyidea}

\section{Defining memory}

The framework distinguishes two kinds of memory:

\textbf{Storage memory}: the model's weights encode facts
(``Paris is the capital of France''). This is a property of the parameters,
not the computation on a specific input.

\textbf{Functional memory}: information from early tokens persists and
influences the output at later layers. This \emph{is} a property of the
computation, and we measure it as \textbf{persistent dependency}:
\[
  \text{Pers}_\tau = \frac{D_\tau}{D_0}
\]
If a token from the beginning of the sequence ($\tau = 0$) still has high
dependency in the final layers, the model is ``remembering'' it functionally.

\section{Defining control}

Control means deliberately changing the model's computation by modifying
the geometry. The polar decomposition gives us two levers:

\begin{itemize}[nosep]
\item \textbf{Metric control}: clamping or modifying the eigenvalues
  of $P^{(l)}$ to change what the layer amplifies.
\item \textbf{Rotation control}: modifying $U^{(l)}$ to change the
  directions of information flow.
\end{itemize}

\subsection{The surprise: metric control dominates}

The research monograph originally predicted that combining rotation and
metric control (``dual control'') would work best, with rotation providing
the primary lever. The experiments showed the opposite:

\begin{finding}
\textbf{Metric control alone is sufficient.}

\begin{center}
\renewcommand{\arraystretch}{1.3}\small
\begin{tabular}{@{}lcc@{}}
\toprule
\textbf{Condition} & \textbf{Dep.\ total} & \textbf{Dep.\ entropy} \\
\midrule
Baseline & $\sim$1.00 & $\sim$2.45 \\
Rotation-only & $\sim$0.91 ($-$9\%) & $\sim$2.33 \\
Metric-only & $\sim$0.52 ($-$48\%) & $\sim$1.21 \\
Dual (both) & $\sim$0.52 ($-$48\%) & $\sim$1.21 \\
\bottomrule
\end{tabular}
\end{center}

Metric-only and dual control produce \emph{identical} results. Rotation
control adds nothing. The reason: the eigenvalue spectrum of $P^{(l)}$
directly controls how gradients flow, while rotation preserves norms and
therefore has little effect on gradient magnitudes.
\end{finding}

This is an important lesson: the theory predicted one thing, the data
showed another, and we updated the theory. Science works.

Functional memory means information from early tokens persists in late

\section{Application 1: Detecting when a model ignores context}

\textbf{The problem.} You provide a document and ask a question about it.

\section{Application 2: Detecting hallucination types}

\textbf{The problem.} A model generates plausible-sounding but incorrect text.

\section{Application 3: Steering model behavior}

\textbf{The problem.} You want to change the model's behavior at inference
time without retraining.

\textbf{The framework view.} Every steering method is a special case of:
\[
  \widetilde{H}^{(l)}_{\text{steered}}
  = U_\Delta \, \widetilde{H}^{(l)} \, P_\Delta
\]
The framework tells you \emph{where} to intervene (layers with both high
curvature and dependency), \emph{what} to change (the metric factor $P$,
based on our control study), and \emph{how much} (guided by the eigenvalue
spectrum).

\begin{lstlisting}[caption={Identifying intervention points for steering}]
import ltg
import numpy as np

result = ltg.analyse(
    "Write a positive review of the restaurant",
    model=model
)

# Find layers where BOTH curvature and dependency are above median
curv = result.curvature_by_layer
dep = result.dependency_profile[:len(curv)]  # align lengths
curv_thresh = np.median(curv)
dep_thresh = np.median(dep)

print("Recommended intervention layers:")
for l in range(len(curv)):
    if curv[l] > curv_thresh and dep[l] > dep_thresh:
        print(f"  Layer {l}: curv={curv[l]:.4f}, dep={dep[l]:.4f}, "
              f"kappa={result.condition_numbers[l]:.1f}")

# These are the "decision points" where steering has maximum effect
# Intervening at low-curvature or low-dependency layers is wasted effort
\end{lstlisting}

\section{The validated findings}

Here is what the framework establishes with cross-model empirical support:

\begin{enumerate}
\item \textbf{Three-phase computation.} Transformers consistently show
  early (setup), middle (processing), and late (decision) computational
  phases, visible in the layer kernel and curvature profile.

\item \textbf{Curvature concentrates in final layers.} The most
  non-trivial computation happens at the end of the network. This
  replicates across architectures and scales.

\item \textbf{Task type is detectable but not dominant.} The type of
  cognitive task affects dependency metrics, but with modest effect
  sizes. The geometry does not cleanly separate ``arithmetic'' from
  ``reasoning.''

\item \textbf{Reasoning depth is invisible to geometry.} The number
  of logical steps in a problem has no measurable geometric signature.
  This was the most important negative result.

\item \textbf{Surface features matter.} Prompt length, formatting,
  and paraphrasing all affect the geometry---you cannot ignore them
  as confounds.

\item \textbf{Metric control dominates rotation control.} For
  intervening on model computation, modifying the stretching component
  is sufficient; changing rotation directions adds nothing measurable.
\end{enumerate}

\chapter{Introduction to Neural Networks and Transformers}
\label{app:nn-transformers}

% (no companion notebook for this appendix)

This appendix provides the background you need to understand what a
transformer language model \emph{does}, so that the geometric tools in the
main text make sense. If you have taken a machine learning course, you can
skim quickly. If not, read carefully---everything in this book builds on
these ideas.

% ──────────────────────────────────────────────────────────────────────────────
\section{From linear models to neural networks}

A \textbf{linear model} predicts an output~$y$ from an input vector~$\mathbf{x}$
using a weighted sum:
\[
  y = \mathbf{w}^\top \mathbf{x} + b
  = w_1 x_1 + w_2 x_2 + \cdots + w_p x_p + b.
\]
The weights~$\mathbf{w}$ and bias~$b$ are learned from data.  This is
powerful---logistic regression, linear SVM, and ordinary least squares are
all linear models---but it can only capture straight-line (hyperplane)
decision boundaries.

A \textbf{neural network} stacks multiple layers of linear transformations
with \emph{nonlinear activation functions} in between:
\[
  \mathbf{h}^{(1)} = \sigma(\mathbf{W}^{(1)} \mathbf{x} + \mathbf{b}^{(1)}),
  \qquad
  \mathbf{h}^{(2)} = \sigma(\mathbf{W}^{(2)} \mathbf{h}^{(1)} + \mathbf{b}^{(2)}),
  \qquad \ldots
\]
where $\sigma$ is a nonlinear function (ReLU, sigmoid, tanh, etc.).  Each
intermediate vector~$\mathbf{h}^{(l)}$ is called a \textbf{hidden state}
at layer~$l$.

\begin{keyidea}
The nonlinearity is what gives neural networks their power.  Without it,
stacking layers would reduce to a single linear transformation---no matter
how many layers you add.
\end{keyidea}

\begin{analogy}
Think of an assembly line.  Each station (layer) receives a work-in-progress
item (hidden state), transforms it, and passes it to the next station.
By the final station, the raw material has become a finished product.
\end{analogy}

\subsection{How networks learn: gradient descent}

A neural network learns by minimizing a \textbf{loss function}~$\mathcal{L}$
that measures how far the network's predictions are from the true answers.
The key algorithm is \textbf{gradient descent}:
\begin{enumerate}[nosep]
\item \textbf{Forward pass:} compute the network's output for a batch of inputs.
\item \textbf{Compute loss:} compare the output to the correct answer.
\item \textbf{Backward pass (backpropagation):} compute
  $\partial \mathcal{L} / \partial \mathbf{W}^{(l)}$ for every layer.
\item \textbf{Update:} nudge each weight in the direction that reduces the loss:
  $\mathbf{W} \leftarrow \mathbf{W} - \eta \, \partial \mathcal{L} / \partial \mathbf{W}$.
\end{enumerate}
The learning rate~$\eta$ controls the step size.  Modern optimizers (Adam,
AdaFactor) adapt~$\eta$ per parameter.

\subsection{Key architectural ideas}

Before transformers, several ideas shaped the field:

\begin{itemize}[nosep]
\item \textbf{Convolutional neural networks (CNNs):} use local filters that
  slide across the input.  Revolutionized image recognition.
\item \textbf{Recurrent neural networks (RNNs):} process sequences one step
  at a time, passing a hidden state forward.  Used for language before
  transformers.
\item \textbf{Long short-term memory (LSTM):} an RNN variant with gating
  mechanisms that help information persist across long sequences.
\end{itemize}

RNNs and LSTMs have a critical limitation: they process tokens
\emph{sequentially}, so they are slow to train on long sequences and
struggle to relate distant tokens.  Transformers solved both problems.

% ──────────────────────────────────────────────────────────────────────────────
\section{The transformer architecture}

The transformer was introduced by Vaswani et al.\ (2017) in ``Attention Is
All You Need.''  Its two key innovations are \textbf{self-attention} and
\textbf{positional encoding}.

\subsection{Tokenization}

Before a transformer sees text, it is split into \textbf{tokens}---subword
units from a fixed vocabulary (typically 32k--128k entries).  For example:

\begin{center}\small
\texttt{"Understanding transformers"} $\;\to\;$
\texttt{["Under", "standing", " transform", "ers"]}
\end{center}

Each token is mapped to a learned \textbf{embedding vector} of dimension~$p$
(commonly $p = 768$ to $p = 8192$).  This embedding is the input to the
first layer.

\subsection{Self-attention: the core mechanism}

Self-attention lets every token look at every other token and decide how
much to ``pay attention'' to it.  For each token~$t$, the model computes:
\begin{itemize}[nosep]
\item A \textbf{query}~$\mathbf{q}_t = \mathbf{W}_Q \mathbf{h}_t$:
  ``What am I looking for?''
\item A \textbf{key}~$\mathbf{k}_t = \mathbf{W}_K \mathbf{h}_t$:
  ``What do I contain?''
\item A \textbf{value}~$\mathbf{v}_t = \mathbf{W}_V \mathbf{h}_t$:
  ``What information do I provide?''
\end{itemize}

The attention score between tokens~$i$ and~$j$ is the scaled dot product of
query and key:
\[
  \text{score}(i, j) = \frac{\mathbf{q}_i^\top \mathbf{k}_j}{\sqrt{d_k}},
\]
where $d_k$ is the dimension of the keys.  These scores are passed through
a softmax to produce attention weights~$\alpha_{ij}$, and the output for
token~$i$ is the weighted sum of all values:
\[
  \text{Attention}(\mathbf{q}_i) = \sum_{j} \alpha_{ij} \, \mathbf{v}_j.
\]

\begin{analogy}
Self-attention is like a meeting where every participant (token) can ask a
question (query) and get answers weighted by relevance (key-query similarity)
from every other participant.  The answers (values) are then combined.
\end{analogy}

\subsection{Multi-head attention}

Instead of computing one set of $(\mathbf{Q}, \mathbf{K}, \mathbf{V})$,
the transformer uses multiple \textbf{attention heads} (typically 12--128),
each with its own learned projections.  Each head can attend to different
aspects of the input.  The outputs are concatenated and projected:
\[
  \text{MultiHead} = \text{Concat}(\text{head}_1, \ldots, \text{head}_H)
  \, \mathbf{W}_O.
\]

\subsection{Feed-forward networks}

After attention, each token's representation passes through a
\textbf{feed-forward network} (FFN)---two linear layers with a nonlinearity
(typically GELU or SiLU):
\[
  \text{FFN}(\mathbf{h}) = \mathbf{W}_2 \, \sigma(\mathbf{W}_1 \mathbf{h}
  + \mathbf{b}_1) + \mathbf{b}_2.
\]
The FFN processes each token \emph{independently}---attention handles
cross-token communication, FFN handles per-token computation.

\subsection{Residual connections and layer normalization}

Each sub-layer (attention, FFN) is wrapped with:
\begin{enumerate}[nosep]
\item A \textbf{residual connection:} $\mathbf{h} \leftarrow \mathbf{h}
  + \text{SubLayer}(\mathbf{h})$.  This lets gradients flow directly
  through the network, enabling training of very deep models.
\item \textbf{Layer normalization:} scales the hidden state to have zero
  mean and unit variance.  Stabilizes training.
\end{enumerate}

\subsection{Positional encoding}

Self-attention is permutation-invariant---it doesn't know the order of
tokens.  \textbf{Positional encodings} inject position information.
Modern models use \textbf{Rotary Position Embedding (RoPE)}, which encodes
relative positions through rotation of query/key vectors.

\subsection{Putting it all together: one transformer layer}

A single transformer layer performs:
\[
  \mathbf{h}' = \text{LayerNorm}(\mathbf{h} + \text{MultiHead}(\mathbf{h})),
\]
\[
  \mathbf{h}^{(l+1)} = \text{LayerNorm}(\mathbf{h}' + \text{FFN}(\mathbf{h}')).
\]
A transformer model stacks $L$ such layers (typically $L = 12$ to $L = 96$).

\begin{keyidea}
\textbf{This is exactly the layer--time grid} from Chapter~1.  The hidden
state $H(l,t)$ at layer~$l$ and token~$t$ is the output of the $l$-th
transformer layer for token~$t$.  The entire framework in this book operates
on these hidden states.
\end{keyidea}

% ──────────────────────────────────────────────────────────────────────────────
\section{Types of transformer models}

Transformers come in three main flavors:

\begin{center}\small
\begin{tabular}{@{}lll@{}}
\toprule
\textbf{Type} & \textbf{Attention} & \textbf{Examples} \\
\midrule
Encoder-only & Bidirectional (sees all tokens) & BERT, RoBERTa \\
Decoder-only & Causal (sees only past tokens) & GPT, LLaMA, Qwen \\
Encoder-decoder & Both & T5, BART \\
\bottomrule
\end{tabular}
\end{center}

This book focuses on \textbf{decoder-only} models (also called
\textbf{causal language models}), because they are the architecture behind
GPT, LLaMA, Qwen, DeepSeek, and essentially all modern large language models.

\chapter{Introduction to GPT and Large Language Models}
\label{app:gpt-llm}

% (no companion notebook for this appendix)

This appendix walks through every step of how a GPT-style large language
model processes text---from raw characters to generated output.
Understanding this pipeline is essential for interpreting the geometric
measurements in the main text, because every quantity we compute
(curvature, dependency, condition numbers) operates on the intermediate
representations produced at each step.

% ──────────────────────────────────────────────────────────────────────────────
\section{The big picture: what GPT does}

GPT (Generative Pre-trained Transformer) is a \textbf{decoder-only}
transformer trained on a single objective: \textbf{predict the next token}.
Given text so far, the model produces a probability distribution over the
entire vocabulary for what comes next.  That is it.  Every capability you
see---question answering, translation, code generation, reasoning---emerges
from this one objective applied to enough data.

The full pipeline from input text to output text has seven stages:

\begin{center}\small
\begin{tabular}{@{}cl@{}}
\toprule
\textbf{Stage} & \textbf{What happens} \\
\midrule
1. Tokenization     & Raw text $\to$ sequence of integer token IDs \\
2. Token embedding  & Token IDs $\to$ dense vectors \\
3. Position encoding & Inject position information into embeddings \\
4. Transformer layers & Repeated self-attention + feed-forward processing \\
5. Final layer norm  & Normalize the output of the last layer \\
6. Language model head & Project hidden states to vocabulary logits \\
7. Decoding          & Convert logits to text (sampling or greedy) \\
\bottomrule
\end{tabular}
\end{center}

We now explain each stage in detail, using the prompt
\texttt{"The capital of France is"} as a running example.

% ──────────────────────────────────────────────────────────────────────────────
\section{Stage 1: Tokenization}

The model does not see characters or words.  It sees \textbf{tokens}---
subword units from a fixed vocabulary, typically 32,000 to 150,000 entries.
The vocabulary is constructed during training using algorithms like
\textbf{Byte-Pair Encoding (BPE)} or \textbf{SentencePiece}.

\begin{keyidea}
Tokenization determines what the model's ``atoms'' of language are.  Common
words become single tokens; rare words are split into subword pieces.  This
balances vocabulary size with sequence length.
\end{keyidea}

\paragraph{Example.} For Qwen2.5-7B, our running example tokenizes as:

\begin{center}\small
\texttt{"The capital of France is"} $\;\to\;$
\texttt{["The", " capital", " of", " France", " is"]}
$\;\to\;$ \texttt{[785, 6620, 315, 9822, 374]}
\end{center}

Each token maps to an integer ID from the vocabulary.  The result is a
sequence of $T = 5$ integers.

\paragraph{How BPE works.} Starting from individual bytes (or characters),
BPE repeatedly merges the most frequent adjacent pair into a new token.
After thousands of merges:
\begin{itemize}[nosep]
\item Common words (\texttt{"the"}, \texttt{"is"}) become single tokens.
\item Rare words (\texttt{"bioluminescence"}) are split:
  \texttt{["bi", "ol", "umin", "escence"]}.
\item Any text can be tokenized---no out-of-vocabulary problem.
\end{itemize}

\begin{lstlisting}[caption={Tokenization in Python}]
from transformers import AutoTokenizer
tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2.5-7B")

text = "The capital of France is"
token_ids = tokenizer.encode(text)
tokens = [tokenizer.decode([tid]) for tid in token_ids]

print(f"Token IDs: {token_ids}")   # [785, 6620, 315, 9822, 374]
print(f"Tokens:    {tokens}")       # ['The', ' capital', ' of', ' France', ' is']
print(f"Vocab size: {tokenizer.vocab_size}")  # 151,665
\end{lstlisting}

\begin{pitfall}
Tokenization is not the same as splitting by spaces.  The token
\texttt{" capital"} includes a leading space.  Some words split
unpredictably.  The number of tokens $T$ determines the width of the
layer--time grid.
\end{pitfall}

% ──────────────────────────────────────────────────────────────────────────────
\section{Stage 2: Token embedding}

Each token ID is mapped to a dense vector of dimension~$p$ (the
\textbf{hidden dimension}) via a learned \textbf{embedding table}
$\mathbf{E} \in \mathbb{R}^{V \times p}$, where $V$ is the vocabulary size:
\[
  \mathbf{e}_t = \mathbf{E}[x_t] \in \mathbb{R}^p.
\]
This is simply a table lookup---row $x_t$ of the matrix~$\mathbf{E}$.

For Qwen2.5-7B, $V = 151{,}665$ and $p = 3{,}584$, so the embedding table
alone contains $V \times p \approx 544$ million parameters.

\begin{analogy}
Think of the embedding table as a dictionary where each word has a
3{,}584-dimensional ``meaning vector'' instead of a text definition.
Semantically similar tokens (``king'' and ``queen'') have vectors that
point in similar directions.
\end{analogy}

After this stage, our 5 tokens become a matrix of shape $(5, 3584)$---five
vectors, one per token.

% ──────────────────────────────────────────────────────────────────────────────
\section{Stage 3: Positional encoding}

Self-attention (Stage~4) is \textbf{permutation-invariant}: it treats the
input as a set, not a sequence.  Without position information, the model
cannot distinguish ``dog bites man'' from ``man bites dog.''

\textbf{Positional encodings} inject position information.  Modern GPT
models use \textbf{Rotary Position Embedding (RoPE)}, which works by
rotating the query and key vectors in attention:
\[
  \mathbf{q}'_t = R(\theta, t) \, \mathbf{q}_t,
  \qquad
  \mathbf{k}'_t = R(\theta, t) \, \mathbf{k}_t,
\]
where $R(\theta, t)$ is a block-diagonal rotation matrix whose angle
increases with position~$t$.  The key property is that the dot product
$\mathbf{q}'_i{}^\top \mathbf{k}'_j$ depends only on the \emph{relative}
distance $|i - j|$, not on absolute positions.

\begin{keyidea}
RoPE encodes \emph{relative} position through rotation.  This connects
directly to the rotation component~$U$ in the polar decomposition
(Chapter~4): some of the rotation we measure is position-dependent
rather than content-dependent.
\end{keyidea}

Older models (GPT-2) used \textbf{learned absolute positional embeddings}:
a separate embedding table $\mathbf{P} \in \mathbb{R}^{T_{\max} \times p}$
added to the token embeddings:
$\mathbf{h}^{(0)}_t = \mathbf{e}_t + \mathbf{P}[t]$.

% ──────────────────────────────────────────────────────────────────────────────
\section{Stage 4: Transformer layers (the core computation)}

This is where the real processing happens.  The model stacks $L$ identical
layers (28 for Qwen2.5-7B), each consisting of two sub-layers:
\textbf{self-attention} and a \textbf{feed-forward network}.

\begin{keyidea}
\textbf{This is the layer--time grid} from Chapter~1.  The hidden state
$H(l,t)$ at layer~$l$ and token~$t$ is the output of the $l$-th transformer
layer for token~$t$.  Everything in this book operates on these hidden
states.
\end{keyidea}

\subsection{Step 4a: Layer normalization (pre-norm)}

Before each sub-layer, the hidden state is normalized:
\[
  \hat{\mathbf{h}} = \text{RMSNorm}(\mathbf{h})
  = \frac{\mathbf{h}}{\text{RMS}(\mathbf{h})} \odot \boldsymbol{\gamma},
  \qquad
  \text{RMS}(\mathbf{h}) = \sqrt{\frac{1}{p}\sum_{i=1}^{p} h_i^2}.
\]
The learnable scale~$\boldsymbol{\gamma}$ has $p$ parameters.  RMSNorm
(used in most modern models) omits the mean centering of the original
LayerNorm.  It stabilizes training by keeping hidden-state magnitudes in
a controlled range.

\subsection{Step 4b: Multi-head self-attention}

Self-attention is the mechanism that lets tokens communicate with each other.
For each token~$t$, the model computes three vectors from the normalized
hidden state $\hat{\mathbf{h}}_t$:

\begin{itemize}[nosep]
\item \textbf{Query} $\mathbf{q}_t = \mathbf{W}_Q \hat{\mathbf{h}}_t$:
  ``What am I looking for?''
\item \textbf{Key} $\mathbf{k}_t = \mathbf{W}_K \hat{\mathbf{h}}_t$:
  ``What do I contain?''
\item \textbf{Value} $\mathbf{v}_t = \mathbf{W}_V \hat{\mathbf{h}}_t$:
  ``What information do I provide if selected?''
\end{itemize}

The weight matrices $\mathbf{W}_Q, \mathbf{W}_K \in \mathbb{R}^{p \times d_k}$
and $\mathbf{W}_V \in \mathbb{R}^{p \times d_v}$ are learned.

\paragraph{Computing attention scores.}  The attention score between
token~$i$ (as query) and token~$j$ (as key) is the scaled dot product:
\[
  a_{ij} = \frac{\mathbf{q}_i^\top \mathbf{k}_j}{\sqrt{d_k}}.
\]
The $\sqrt{d_k}$ scaling prevents the dot products from becoming too large,
which would push the softmax into saturation.

\paragraph{Causal masking.}  In a GPT model, token~$t$ must not attend to
future tokens $t+1, t+2, \ldots$.  This is enforced by setting
$a_{ij} = -\infty$ for all $j > i$ before the softmax:
\[
  \alpha_{ij} = \frac{\exp(a_{ij})}{\sum_{k \leq i} \exp(a_{ik})},
  \qquad j \leq i.
\]
The resulting attention weights $\alpha_{ij}$ form a lower-triangular matrix.

\paragraph{Weighted aggregation.}  The attention output for token~$i$ is
the weighted sum of all value vectors it can see:
\[
  \text{Attn}(\mathbf{q}_i) = \sum_{j \leq i} \alpha_{ij} \, \mathbf{v}_j.
\]

\paragraph{Multiple heads.}  Instead of one set of $(\mathbf{Q}, \mathbf{K},
\mathbf{V})$, the model uses $H$ parallel \textbf{attention heads}
(e.g., $H = 28$ for Qwen2.5-7B), each with its own projections and operating
on a slice of the hidden dimension ($d_k = p / H$).  Each head can
specialize in different types of relationships (syntactic, semantic,
positional).  The outputs are concatenated and projected:
\[
  \text{MultiHead}(\mathbf{h}_t) =
  \text{Concat}(\text{head}_1, \ldots, \text{head}_H) \, \mathbf{W}_O,
\]
where $\mathbf{W}_O \in \mathbb{R}^{p \times p}$ is the output projection.

\begin{analogy}
Multi-head attention is like a committee meeting with $H$ sub-committees.
Each sub-committee (head) investigates a different aspect of the question.
Their reports are concatenated into a single summary.
\end{analogy}

\paragraph{Grouped-Query Attention (GQA).}  Modern models like Qwen2.5
and LLaMA-3 use \textbf{GQA}: multiple query heads share a single key-value
head.  For example, Qwen2.5-7B has 28 query heads but only 4 key-value
heads (each shared by 7 query heads).  This reduces the KV-cache size by
$\sim 7\times$ with minimal quality loss.

\subsection{Step 4c: Residual connection (attention)}

The attention output is \emph{added} to the original hidden state:
\[
  \mathbf{h}'_t = \mathbf{h}_t + \text{MultiHead}(\text{RMSNorm}(\mathbf{h}_t)).
\]
This \textbf{residual connection} is critical: it creates a ``highway'' for
gradients during training and ensures that each layer makes an
\emph{incremental update} rather than a complete replacement.

\begin{keyidea}
Residual connections explain why the layer kernel (Chapter~3) shows high
similarity between adjacent layers: each layer only adds a small perturbation
to the previous representation.  Without residuals, consecutive layers
could produce wildly different representations.
\end{keyidea}

\subsection{Step 4d: Feed-forward network (FFN)}

After attention, each token is processed \emph{independently} by a
feed-forward network.  Modern GPT models use the \textbf{SwiGLU} variant:
\[
  \text{FFN}(\mathbf{h}) =
  \bigl(\text{SiLU}(\mathbf{W}_1 \mathbf{h}) \odot \mathbf{W}_3 \mathbf{h}\bigr)
  \, \mathbf{W}_2,
\]
where $\text{SiLU}(x) = x \cdot \sigma(x)$ is a smooth activation,
$\odot$ is element-wise multiplication,
$\mathbf{W}_1, \mathbf{W}_3 \in \mathbb{R}^{p \times d_{\text{ff}}}$, and
$\mathbf{W}_2 \in \mathbb{R}^{d_{\text{ff}} \times p}$.

The intermediate dimension $d_{\text{ff}}$ is typically $\sim 2.7 \times p$
(e.g., 18{,}944 for Qwen2.5-7B with $p = 3{,}584$).  The FFN accounts for
about two-thirds of the model's parameters.

\begin{keyidea}
Self-attention handles \emph{cross-token} communication (which tokens
should influence each other).  The FFN handles \emph{per-token} computation
(transforming each token's representation independently).  In the geometric
framework, the FFN primarily contributes to the \textbf{stretching}
(metric factor~$P$) in the polar decomposition, because it amplifies some
directions and compresses others within each token's representation.
\end{keyidea}

\subsection{Step 4e: Residual connection (FFN)}

The FFN output is again added via a residual connection:
\[
  \mathbf{h}^{(l+1)}_t = \mathbf{h}'_t + \text{FFN}(\text{RMSNorm}(\mathbf{h}'_t)).
\]
The vector $\mathbf{h}^{(l+1)}_t$ is the hidden state at layer $l+1$ for
token~$t$---the next row in the layer--time grid.

\subsection{Summary: one complete layer}

Putting it all together, one transformer layer computes:
\begin{align*}
  \mathbf{h}'_t &= \mathbf{h}^{(l)}_t + \text{MultiHead}\bigl(\text{RMSNorm}(\mathbf{h}^{(l)}_t)\bigr)
  && \text{(attention + residual)} \\
  \mathbf{h}^{(l+1)}_t &= \mathbf{h}'_t + \text{FFN}\bigl(\text{RMSNorm}(\mathbf{h}'_t)\bigr)
  && \text{(FFN + residual)}
\end{align*}
This is repeated $L$ times.  For Qwen2.5-7B ($L = 28$), the input embedding
passes through 28 layers of this computation, producing a stack of 28 hidden
states per token---the columns of the layer--time grid.

\begin{lstlisting}[caption={Tracing the shapes through one layer (pseudocode)}]
# Input: h of shape (T, p) = (5, 3584)

# Attention sub-layer
h_norm = rms_norm(h)                    # (5, 3584)
Q = h_norm @ W_Q                        # (5, d_k*H) -> split into H heads
K = h_norm @ W_K                        # (5, d_k*H_kv) -> shared KV heads
V = h_norm @ W_V                        # (5, d_v*H_kv)
attn_out = multi_head_attention(Q, K, V) # (5, 3584)
h = h + attn_out                         # residual -> (5, 3584)

# FFN sub-layer
h_norm = rms_norm(h)                     # (5, 3584)
gate = silu(h_norm @ W1)                 # (5, 18944)
up = h_norm @ W3                         # (5, 18944)
ffn_out = (gate * up) @ W2              # (5, 3584)
h = h + ffn_out                          # residual -> (5, 3584)

# Output: h of shape (T, p) = (5, 3584) -- same shape, new values
\end{lstlisting}

% ──────────────────────────────────────────────────────────────────────────────
\section{Stage 5: Final layer normalization}

After the last transformer layer, one more RMSNorm is applied:
\[
  \mathbf{h}^{(\text{final})}_t = \text{RMSNorm}(\mathbf{h}^{(L)}_t).
\]
This ensures the representations entering the language model head have
consistent scale.

% ──────────────────────────────────────────────────────────────────────────────
\section{Stage 6: Language model head}

The \textbf{language model head} (also called the \textbf{unembedding})
projects each token's final hidden state onto the vocabulary:
\[
  \boldsymbol{\ell}_t = \mathbf{W}_{\text{head}} \, \mathbf{h}^{(\text{final})}_t
  \in \mathbb{R}^V,
\]
where $\mathbf{W}_{\text{head}} \in \mathbb{R}^{V \times p}$.  Many models
\emph{tie} this matrix to the transposed embedding table:
$\mathbf{W}_{\text{head}} = \mathbf{E}^\top$, so the same vectors that
define token meanings are also used to predict tokens.

The vector $\boldsymbol{\ell}_t$ is called the \textbf{logits}---raw
unnormalized scores for each vocabulary entry.  A softmax converts them to
probabilities:
\[
  P(x_{t+1} = w \mid x_1, \ldots, x_t) =
  \frac{\exp(\ell_{t,w})}{\sum_{w'=1}^{V} \exp(\ell_{t,w'})}.
\]

\begin{keyidea}
The \textbf{dependency} $D_l = \|\partial s / \partial \tilde{H}^{(l)}\|$
from Chapter~7 measures how much each layer's hidden state contributes to
these logits.  High dependency at layer~$l$ means that layer~$l$'s
computation materially affects the next-token prediction.
\end{keyidea}

% ──────────────────────────────────────────────────────────────────────────────
\section{Stage 7: Decoding}

Given the probability distribution $P(x_{t+1} \mid \ldots)$, we need to
select an actual token.  This is the \textbf{decoding} step.

\subsection{Greedy decoding}

Pick the highest-probability token:
$x_{t+1} = \arg\max_w P(w \mid x_1, \ldots, x_t)$.
Fast and deterministic, but can produce repetitive, ``safe'' text.

\subsection{Sampling with temperature}

Scale the logits before softmax:
$P_\tau(w) \propto \exp(\ell_w / \tau)$.
\begin{itemize}[nosep]
\item $\tau < 1$: sharper distribution, more ``confident'' (closer to greedy).
\item $\tau = 1$: unmodified probabilities.
\item $\tau > 1$: flatter distribution, more ``creative'' (more randomness).
\end{itemize}

\subsection{Top-$k$ sampling}

Restrict sampling to the $k$ most likely tokens, setting all other
probabilities to zero.  For example, top-$k = 50$ means only the 50
highest-probability tokens are considered.

\subsection{Top-$p$ (nucleus) sampling}

Instead of a fixed $k$, include the smallest set of tokens whose cumulative
probability exceeds $p$ (e.g., $p = 0.9$).  This adapts the candidate set
to the model's confidence: when the model is certain, very few tokens
qualify; when uncertain, many do.

\subsection{Autoregressive generation loop}

Generation proceeds one token at a time:
\begin{enumerate}[nosep]
\item \textbf{Prompt encoding:} run the full prompt through all $L$ layers.
  Cache the key and value matrices (\textbf{KV-cache}).
\item \textbf{Sample next token:} apply the LM head to the last token's
  final hidden state, decode one token.
\item \textbf{Append and continue:} feed the new token through all layers
  (using the KV-cache to avoid recomputing past tokens).
\item \textbf{Repeat} until the model emits an end-of-sequence token or
  a maximum length is reached.
\end{enumerate}

\begin{lstlisting}[caption={Autoregressive generation (simplified)}]
# Prompt encoding (prefill)
token_ids = tokenizer.encode("The capital of France is")
hidden_states, kv_cache = model.forward(token_ids)

# Generation loop
for step in range(max_new_tokens):
    logits = lm_head(hidden_states[:, -1, :])  # last token only
    probs = softmax(logits / temperature)
    next_token = sample(probs, top_p=0.9)

    if next_token == eos_token_id:
        break

    # Feed just the new token, reusing cached K,V
    hidden_states, kv_cache = model.forward(
        [next_token], kv_cache=kv_cache
    )
    token_ids.append(next_token)

print(tokenizer.decode(token_ids))
# "The capital of France is Paris."
\end{lstlisting}

\begin{keyidea}
The \textbf{KV-cache} is why generation cost scales linearly with sequence
length rather than quadratically.  At each step, only the new token's
queries need to attend to all cached keys/values---the past keys and values
are not recomputed.
\end{keyidea}

% ──────────────────────────────────────────────────────────────────────────────
\section{The complete forward pass: a worked example}

Let us trace the entire pipeline for our running example:

\begin{enumerate}
\item \textbf{Input:} \texttt{"The capital of France is"}

\item \textbf{Tokenize:} $[785, 6620, 315, 9822, 374]$ --- 5 tokens.

\item \textbf{Embed:} Look up 5 rows from $\mathbf{E}$
  $\to$ matrix of shape $(5, 3584)$.

\item \textbf{Position:} Apply RoPE rotations to Q and K at each layer
  (positions 0--4).

\item \textbf{Layer 1:}
  \begin{itemize}[nosep]
  \item RMSNorm $\to$ compute Q, K, V $\to$ causal attention $\to$
    residual add.
  \item RMSNorm $\to$ SwiGLU FFN $\to$ residual add.
  \item Output: $(5, 3584)$---the hidden state at layer~1.
  \end{itemize}

\item \textbf{Layers 2--28:} repeat the same computation.  Each layer reads
  the previous layer's output and writes a new $(5, 3584)$ matrix.

\item \textbf{Final RMSNorm:} normalize the output of layer~28.

\item \textbf{LM head:} project the last token's hidden state
  ($\mathbf{h}^{(28)}_5 \in \mathbb{R}^{3584}$) to logits
  $\in \mathbb{R}^{151665}$.

\item \textbf{Softmax:} convert logits to probabilities.  The
  highest-probability token is likely \texttt{"Paris"} (token ID for
  ``~Paris'').

\item \textbf{Decode:} output \texttt{"Paris"} and optionally continue
  generating.
\end{enumerate}

The \textbf{layer--time grid} from Chapter~1 is the $28 \times 5$ matrix
of hidden states produced in steps 5--6.  The \texttt{ltg.analyse()} function
extracts these hidden states and computes all geometric quantities on them.

% ──────────────────────────────────────────────────────────────────────────────
\section{Training: how GPT learns}

\subsection{The training objective}

GPT is trained on \textbf{next-token prediction}.  Given a sequence
$(x_1, \ldots, x_T)$ from the training corpus, the loss is:
\[
  \mathcal{L} = -\sum_{t=1}^{T} \log P(x_t \mid x_1, \ldots, x_{t-1}).
\]
Thanks to causal masking, all $T$ predictions are computed in a single
forward pass---making training highly parallelizable (unlike RNNs, which
must process tokens sequentially).

\subsection{Pre-training}

The model is trained on a massive text corpus (trillions of tokens) using
gradient descent with the Adam optimizer.  This \textbf{pre-training} phase
gives the model broad language understanding, factual knowledge, and basic
reasoning abilities.

\subsection{Fine-tuning and alignment}

After pre-training, the model is further refined:
\begin{enumerate}[nosep]
\item \textbf{Supervised fine-tuning (SFT):} train on curated
  (instruction, response) pairs to teach the model to follow instructions.
\item \textbf{RLHF / DPO:} use human preference data to align the model's
  outputs with human expectations.  \textbf{RLHF} (Reinforcement Learning
  from Human Feedback) trains a reward model and optimizes against it.
  \textbf{DPO} (Direct Preference Optimization) skips the reward model
  and directly optimizes on preference pairs.
\end{enumerate}

\section{Why study model internals?}

LLMs are powerful but opaque.  When a model hallucinates, refuses a valid
request, or gives a subtly wrong answer, \textbf{we cannot tell why from the
output alone}.  Understanding the internal computation is essential for:

\begin{itemize}[nosep]
\item \textbf{Debugging:} Why did the model ignore the context I provided?
  (Chapter~\ref{chap:applications}, Section~1.)
\item \textbf{Trust:} Is the model retrieving a fact or confabulating?
  (Chapter~\ref{chap:applications}, Section~2.)
\item \textbf{Steering:} Can I modify the model's behavior at specific layers
  without retraining?  (Chapter~\ref{chap:reasoning}, Section~3.)
\item \textbf{Efficiency:} Which layers can be pruned or quantized without
  losing important computation?  (Chapter~\ref{chap:dependency}.)
\item \textbf{Safety:} How do we ensure models behave as intended when
  deployed in high-stakes settings?
\end{itemize}

\begin{keyidea}
This book gives you concrete, geometric tools for all of these tasks.  The
hidden states are not a black box---they have measurable structure, and that
structure tells you what the model is doing.
\end{keyidea}

\section{Vectors, norms, and inner products}

A \textbf{vector} $\mathbf{h} \in \mathbb{R}^p$ is an ordered list of $p$
real numbers.  In this book, every hidden state at position $(l, t)$ on the
layer-time grid is such a vector (Chapter~\ref{chap:intro}).

\begin{keyidea}
Think of each hidden state as a point in $p$-dimensional space.  Geometric
tools---distance, angle, projection---become tools for understanding what
the model is doing at each step.
\end{keyidea}

\paragraph{Euclidean norm.}
The \textbf{length} (or $\ell_2$ norm) of a vector is
\[
  \|\mathbf{h}\| \;=\; \sqrt{h_1^2 + h_2^2 + \cdots + h_p^2}
  \;=\; \sqrt{\mathbf{h}^\top \mathbf{h}}.
\]
We use norms throughout to measure how ``far'' hidden states are from each
other or from the origin.

\paragraph{Inner product.}
The \textbf{dot product} (or inner product) of two vectors is
\[
  \mathbf{a}^\top \mathbf{b} \;=\; \sum_{i=1}^{p} a_i \, b_i.
\]
The dot product encodes both magnitude and alignment:
$\mathbf{a}^\top \mathbf{b} = \|\mathbf{a}\|\,\|\mathbf{b}\|\cos\theta$,
where $\theta$ is the angle between the two vectors.  When
$\mathbf{a}^\top\mathbf{b} = 0$ the vectors are \textbf{orthogonal}
(perpendicular).

\paragraph{Cosine similarity.}
Normalizing the dot product gives a scale-invariant similarity measure:
\[
  \cos\theta \;=\;
  \frac{\mathbf{a}^\top \mathbf{b}}{\|\mathbf{a}\|\,\|\mathbf{b}\|}.
\]
The layer kernel in Chapter~\ref{chap:kernels} averages cosine similarities
across token positions to compare entire layers.

% ──────────────────────────────────────────────────────────────────────────────
\section{Matrices and linear transformations}

A \textbf{matrix} $A \in \mathbb{R}^{m \times n}$ is a rectangular array of
numbers.  Multiplying a matrix by a vector, $\mathbf{y} = A\mathbf{x}$,
defines a \textbf{linear transformation}: it maps every input vector
$\mathbf{x} \in \mathbb{R}^n$ to an output $\mathbf{y} \in \mathbb{R}^m$,
and it preserves addition and scaling.

In this book, the central matrix is the \textbf{layer transition operator}
$T^{(l)}$ (Chapter~\ref{chap:operators}), the best linear map from hidden
states at layer $l$ to hidden states at layer $l+1$:
\[
  \mathbf{h}^{(l+1)} \;\approx\; T^{(l)}\,\mathbf{h}^{(l)}.
\]

\paragraph{Transpose.}
The \textbf{transpose} $A^\top$ flips rows and columns:
$(A^\top)_{ij} = A_{ji}$.  For vectors, $\mathbf{h}^\top$ turns a column
vector into a row vector, making the dot product
$\mathbf{a}^\top\mathbf{b}$ a $1\times 1$ matrix multiplication.

\paragraph{Matrix inverse.}
If $A$ is square and non-singular, its \textbf{inverse} $A^{-1}$ satisfies
$A^{-1}A = AA^{-1} = I$ (the identity matrix).  The transition operator is
computed via least squares, which uses the \textbf{pseudoinverse} when an
exact inverse does not exist (Chapter~\ref{chap:operators}).

\paragraph{Trace and determinant.}
The \textbf{trace} $\operatorname{tr}(A) = \sum_i A_{ii}$ is the sum of
diagonal entries.  The \textbf{determinant} $\det(A)$ measures the volume
scaling factor of the transformation.  For the stretch matrix $P$:
\[
  \operatorname{tr}(P) = \sum_i \sigma_i, \qquad
  \det(P) = \prod_i \sigma_i,
\]
where $\sigma_i$ are the eigenvalues.  A near-zero determinant signals
near-singularity---the layer is collapsing some directions.

% ──────────────────────────────────────────────────────────────────────────────
\section{Special matrix types}
\label{sec:special-matrices}

Several matrix types play starring roles in the geometric framework.

\paragraph{Symmetric matrices.}
A matrix $A$ is \textbf{symmetric} if $A = A^\top$.  Symmetric matrices
have real eigenvalues and orthogonal eigenvectors (the \emph{spectral
theorem}).  The covariance matrix $\Sigma$ and the stretch factor $P$ are
both symmetric.

\paragraph{Positive definite and positive semidefinite.}
A symmetric matrix $A$ is \textbf{positive definite} (PD) if
$\mathbf{x}^\top A\, \mathbf{x} > 0$ for every non-zero $\mathbf{x}$.
It is \textbf{positive semidefinite} (PSD) if $\geq 0$ replaces $> 0$.
\begin{itemize}[nosep]
  \item PD means all eigenvalues are strictly positive---every direction is
    stretched, never collapsed to zero.
  \item The covariance matrix $\Sigma$ is always PSD.
  \item The stretch factor $P$ in the polar decomposition is PSD by
    construction (Chapter~\ref{chap:operators}).
\end{itemize}

\paragraph{Orthogonal matrices.}
A square matrix $U$ is \textbf{orthogonal} if $U^\top U = I$.
Equivalently, its columns form an orthonormal basis.  Orthogonal matrices
preserve lengths and angles:
\[
  \|U\mathbf{x}\| = \|\mathbf{x}\| \quad \text{for all } \mathbf{x}.
\]
Geometrically, orthogonal transformations are \emph{rotations} (if
$\det U = +1$) or \emph{reflections} (if $\det U = -1$).  In the polar
decomposition $T = UP$, the factor $U$ is orthogonal: it redirects
information without changing magnitudes (Chapter~\ref{chap:operators}).

\paragraph{The identity matrix.}
The $k \times k$ identity matrix $I$ has ones on the diagonal and zeros
elsewhere.  When $P = I$, the layer performs no stretching; when
$\|P - I\|_F$ is large, the layer's stretching deviates significantly from
``do nothing'' (Chapter~\ref{chap:curvature}).

% ──────────────────────────────────────────────────────────────────────────────
\section{Eigenvalues and eigenvectors}
\label{sec:eigen}

An \textbf{eigenvector} of a square matrix $A$ is a non-zero vector
$\mathbf{v}$ that is only scaled (not rotated) by $A$:
\[
  A\mathbf{v} = \lambda\,\mathbf{v},
\]
where the scalar $\lambda$ is the corresponding \textbf{eigenvalue}.

\begin{analogy}
Imagine pushing a door.  Most pushes change both the door's direction and
speed.  An eigenvector is the special direction where the door only speeds
up or slows down---it does not turn.  The eigenvalue tells you the
speed-up factor.
\end{analogy}

\paragraph{Spectral theorem (for symmetric matrices).}
If $A$ is $k\times k$ and symmetric, it has $k$ real eigenvalues
$\lambda_1 \geq \lambda_2 \geq \cdots \geq \lambda_k$ with corresponding
orthonormal eigenvectors $\mathbf{v}_1,\ldots,\mathbf{v}_k$.  We can write:
\[
  A = V \Lambda V^\top, \qquad
  V = [\mathbf{v}_1 \mid \cdots \mid \mathbf{v}_k], \qquad
  \Lambda = \operatorname{diag}(\lambda_1,\ldots,\lambda_k).
\]
This is the \textbf{eigendecomposition} (or spectral decomposition).

\paragraph{Why it matters for this book.}
\begin{itemize}[nosep]
  \item The eigenvectors of the covariance matrix $\Sigma$ are the
    principal components used in whitening (Chapter~\ref{chap:whitening}).
  \item The eigenvalues of the stretch factor $P^{(l)}$ reveal which
    directions each layer amplifies or compresses
    (Chapter~\ref{chap:operators}).
  \item The eigenvalue distribution of $P$ determines the effective rank
    and condition number---our key selectivity metrics
    (Chapter~\ref{chap:operators}).
\end{itemize}

% ──────────────────────────────────────────────────────────────────────────────
\section{Singular value decomposition (SVD)}
\label{sec:svd}

The eigendecomposition requires a square symmetric matrix.  The
\textbf{singular value decomposition} generalizes to \emph{any}
$m\times n$ matrix:
\[
  A = U_{\mathrm{svd}} \,\Sigma_{\mathrm{svd}}\, V_{\mathrm{svd}}^\top,
\]
where:
\begin{itemize}[nosep]
  \item $U_{\mathrm{svd}} \in \mathbb{R}^{m \times m}$ is orthogonal
    (left singular vectors),
  \item $\Sigma_{\mathrm{svd}} \in \mathbb{R}^{m \times n}$ is diagonal
    with non-negative \textbf{singular values}
    $\sigma_1 \geq \sigma_2 \geq \cdots \geq 0$,
  \item $V_{\mathrm{svd}} \in \mathbb{R}^{n \times n}$ is orthogonal
    (right singular vectors).
\end{itemize}

\begin{keyidea}
Every linear transformation can be decomposed into three steps: (1) rotate
the input ($V_{\mathrm{svd}}^\top$), (2) stretch along coordinate axes
($\Sigma_{\mathrm{svd}}$), (3) rotate the output ($U_{\mathrm{svd}}$).
The singular values tell you how much stretching happens in each direction.
\end{keyidea}

\paragraph{Connection to eigenvalues.}
For a symmetric PSD matrix $A$, the singular values equal the eigenvalues.
For a general matrix, the singular values of $A$ are the square roots of
the eigenvalues of $A^\top A$.

\paragraph{Use in this book.}
SVD is the computational engine behind two core operations:
\begin{enumerate}[nosep]
  \item \textbf{PCA/whitening} (Chapter~\ref{chap:whitening}): we compute
    the SVD of the centered hidden-state matrix to extract the top-$k$
    principal components.
  \item \textbf{Polar decomposition} (Chapter~\ref{chap:operators}): we factor
    the transition operator $T = UP$ by computing $T = U_{\mathrm{svd}}
    \Sigma_{\mathrm{svd}} V_{\mathrm{svd}}^\top$ and setting
    $U = U_{\mathrm{svd}} V_{\mathrm{svd}}^\top$ and
    $P = V_{\mathrm{svd}}\, \Sigma_{\mathrm{svd}}\, V_{\mathrm{svd}}^\top$.
\end{enumerate}

% ──────────────────────────────────────────────────────────────────────────────
\section{Covariance and PCA}
\label{sec:covariance-pca}

Given $N$ data points $\mathbf{x}_1,\ldots,\mathbf{x}_N \in \mathbb{R}^p$
with mean $\bar{\mathbf{x}}$, the \textbf{covariance matrix} is
\[
  \Sigma = \frac{1}{N-1}\sum_{i=1}^{N}
  (\mathbf{x}_i - \bar{\mathbf{x}})
  (\mathbf{x}_i - \bar{\mathbf{x}})^\top
  \;\in\; \mathbb{R}^{p \times p}.
\]
Entry $\Sigma_{jk}$ measures how much dimensions $j$ and $k$ co-vary.

\paragraph{Principal Component Analysis (PCA).}
PCA finds the directions of maximum variance:
\begin{enumerate}[nosep]
  \item Center the data: $\tilde{\mathbf{x}}_i = \mathbf{x}_i -
    \bar{\mathbf{x}}$.
  \item Compute $\Sigma$ (or use SVD of the centered data matrix).
  \item Take the top-$k$ eigenvectors $V_k = [\mathbf{v}_1,\ldots,
    \mathbf{v}_k]$ of $\Sigma$.
  \item Project: $\mathbf{z}_i = V_k^\top \tilde{\mathbf{x}}_i \in
    \mathbb{R}^k$.
\end{enumerate}
This reduces the dimension from $p$ to $k$ while preserving as much
variance as possible.

\paragraph{Whitening.}
PCA projection followed by rescaling each component to unit variance:
\[
  W = V_k \operatorname{diag}(\lambda_1^{-1/2},\ldots,\lambda_k^{-1/2}),
  \qquad \mathbf{z}_i = W^\top (\mathbf{x}_i - \bar{\mathbf{x}}).
\]
After whitening, the covariance of the $\mathbf{z}_i$ is $I_k$---all
directions have unit variance and zero correlation.  This is the essential
preprocessing step that makes geometric comparisons valid across layers
(Chapter~\ref{chap:whitening}).

\begin{pitfall}
Without whitening, distances and angles in the raw hidden-state space are
dominated by a few high-variance directions.  Two vectors might look
similar only because they share a large component along the top principal
axis, even though they differ in every other direction.  Whitening removes
this bias.
\end{pitfall}

% ──────────────────────────────────────────────────────────────────────────────
\section{Polar decomposition}
\label{sec:polar}

Every invertible matrix $T$ can be uniquely factored as
\[
  T = U P,
\]
where $U$ is orthogonal and $P$ is symmetric positive definite.  This is
the matrix analog of writing a complex number in polar form
$z = e^{i\theta} \cdot r$ (Chapter~\ref{chap:operators}).

\begin{analogy}
Imagine a block of clay.  Applying $T$ to the clay is equivalent to first
\emph{stretching} it ($P$) and then \emph{rotating} it ($U$).
Separating these two actions lets you ask: ``how much did the model
reshape information?'' ($P$) versus ``which direction did the model point
it?'' ($U$).
\end{analogy}

\paragraph{Computing it via SVD.}
Given $T = U_{\mathrm{svd}}\,\Sigma_{\mathrm{svd}}\,V_{\mathrm{svd}}^\top$:
\[
  U = U_{\mathrm{svd}}\, V_{\mathrm{svd}}^\top, \qquad
  P = V_{\mathrm{svd}}\, \Sigma_{\mathrm{svd}}\, V_{\mathrm{svd}}^\top.
\]
The eigenvalues of $P$ are the singular values of $T$.

% ──────────────────────────────────────────────────────────────────────────────
\section{Condition number and effective rank}
\label{sec:condition-rank}

Two scalar summaries of the eigenvalue spectrum of $P$ appear throughout
the book.

\paragraph{Condition number.}
\[
  \kappa = \frac{\sigma_1}{\sigma_k},
\]
where $\sigma_1$ and $\sigma_k$ are the largest and smallest eigenvalues
of $P$.  A large $\kappa$ means the layer amplifies some directions far
more than others---it is making a \emph{selective} transformation.  A
$\kappa$ near 1 means all directions are treated roughly equally
(Chapter~\ref{chap:operators}).

\paragraph{Effective rank.}
Let $\hat\sigma_i = \sigma_i / \sum_j \sigma_j$ be the normalized
eigenvalue distribution.  The \textbf{effective rank} is:
\[
  \mathrm{erank} = \exp\!\Bigl(-\sum_{i=1}^{k} \hat\sigma_i
  \log \hat\sigma_i\Bigr) = \exp(H(\hat\sigma)),
\]
where $H$ is the Shannon entropy.  This measures how many directions are
\emph{actively} used:
\begin{itemize}[nosep]
  \item If all $\sigma_i$ are equal, $\mathrm{erank} = k$ (full rank).
  \item If one $\sigma_i$ dominates, $\mathrm{erank} \approx 1$ (all
    computation in a single direction).
\end{itemize}
Low effective rank means the layer concentrates its computation in a
low-dimensional subspace---a fingerprint of focused, selective processing
(Chapters \ref{chap:operators} and~\ref{chap:experiments}).

% ──────────────────────────────────────────────────────────────────────────────
\section{Norms for matrices}

We use two matrix norms in this book.

\paragraph{Frobenius norm.}
\[
  \|A\|_F = \sqrt{\sum_{i,j} A_{ij}^2}
  = \sqrt{\operatorname{tr}(A^\top A)}
  = \sqrt{\sum_i \sigma_i^2},
\]
where $\sigma_i$ are the singular values.  Think of it as the Euclidean
norm applied to the matrix treated as one long vector.  The curvature
measure in Chapter~\ref{chap:curvature} uses $\|\Omega^{(l,t)} - I\|_F$
to quantify how much the commutator deviates from the identity.

\paragraph{Spectral norm (operator norm).}
\[
  \|A\|_2 = \sigma_1 = \max_{\|\mathbf{x}\|=1} \|A\mathbf{x}\|.
\]
This is the largest singular value---the maximum stretching the matrix can
apply.  The condition number is $\kappa = \|A\|_2 \cdot \|A^{-1}\|_2$.

% ──────────────────────────────────────────────────────────────────────────────
\section{Gram matrices and kernels}

A \textbf{Gram matrix} stores all pairwise inner products of a set of
vectors.  Given $n$ vectors $\mathbf{x}_1,\ldots,\mathbf{x}_n$:
\[
  G_{ij} = \mathbf{x}_i^\top \mathbf{x}_j.
\]

In Chapter~\ref{chap:kernels}, we build two Gram matrices:
\begin{itemize}[nosep]
  \item \textbf{Layer kernel} $K^{\mathrm{layer}}(i,j)$: average cosine
    similarity between token vectors at layers $i$ and $j$.  It answers:
    ``how similar are the representations at these two layers?''
  \item \textbf{Temporal kernel} $K^{\mathrm{token}}(s,t)$: average cosine
    similarity between tokens $s$ and $t$ across all layers.  It answers:
    ``how similarly does the model process these two tokens?''
\end{itemize}
Block-diagonal structure in the layer kernel reveals the three computational
phases (embedding, processing, prediction) described in
Chapter~\ref{chap:kernels}.

% ──────────────────────────────────────────────────────────────────────────────
\section{Gradients and the Jacobian}

\paragraph{Gradient.}
For a scalar-valued function $f(\mathbf{x})$, the \textbf{gradient}
$\nabla_{\mathbf{x}} f \in \mathbb{R}^p$ points in the direction of
steepest increase and has magnitude equal to the rate of change in that
direction:
\[
  \nabla_{\mathbf{x}} f =
  \begin{bmatrix}
    \partial f / \partial x_1 \\ \vdots \\ \partial f / \partial x_p
  \end{bmatrix}.
\]

\paragraph{Jacobian.}
For a vector-valued function $\mathbf{f}: \mathbb{R}^n \to \mathbb{R}^m$,
the \textbf{Jacobian} $J \in \mathbb{R}^{m \times n}$ collects all partial
derivatives: $J_{ij} = \partial f_i / \partial x_j$.

\paragraph{Use in this book.}
The dependency measure in Chapter~\ref{chap:dependency} computes
\[
  D_l = \left\|\frac{\partial s}{\partial H^{(l)}}\right\|_F,
\]
where $s$ is a scalar output (e.g., the logit of the predicted token) and
$H^{(l)}$ is the hidden-state matrix at layer $l$.  This is the Frobenius
norm of a Jacobian: it measures how much the output would change if the
hidden states at layer $l$ were perturbed.  Large $D_l$ means layer $l$
has high influence on the prediction.

% ──────────────────────────────────────────────────────────────────────────────
\section{Least squares}

Given an overdetermined system $A\mathbf{x} \approx \mathbf{b}$ (more
equations than unknowns), the \textbf{least-squares solution} minimizes the
residual:
\[
  \hat{\mathbf{x}} = \arg\min_{\mathbf{x}} \|A\mathbf{x} - \mathbf{b}\|^2.
\]
When $A$ has full column rank, the closed-form solution is:
\[
  \hat{\mathbf{x}} = (A^\top A)^{-1} A^\top \mathbf{b}.
\]
The matrix $(A^\top A)^{-1} A^\top$ is the \textbf{pseudoinverse} of $A$.

\paragraph{Use in this book.}
The transition operator is the least-squares fit
$T^{(l)} = \arg\min_T \|H^{(l)}T - H^{(l+1)}\|_F^2$---the best linear
map from one layer's hidden states to the next
(Chapter~\ref{chap:operators}).  In code this is computed by
\texttt{torch.linalg.lstsq}.

% ──────────────────────────────────────────────────────────────────────────────
\section{Mahalanobis distance}
\label{sec:mahalanobis}

The \textbf{Mahalanobis distance} between a point $\mathbf{x}$ and a
distribution with mean $\boldsymbol{\mu}$ and covariance $\Sigma$ is
\[
  d_M(\mathbf{x}, \boldsymbol{\mu}) =
  \sqrt{(\mathbf{x} - \boldsymbol{\mu})^\top \Sigma^{-1}
  (\mathbf{x} - \boldsymbol{\mu})}.
\]
It accounts for correlations between dimensions: two points that are close
in Euclidean distance may be far apart in Mahalanobis distance if they
differ along a low-variance direction.

\begin{keyidea}
After PCA whitening, the covariance becomes $I$, so the Mahalanobis
distance \emph{equals} the ordinary Euclidean distance.  This is exactly
why whitening is so powerful: it turns the complicated, correlation-aware
Mahalanobis distance into plain Euclidean distance, which is much easier
to compute and interpret (Chapter~\ref{chap:whitening}).
\end{keyidea}
